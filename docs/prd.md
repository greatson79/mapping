## 1. 제품 개요

### 1.1 제품 설명

네이버 지도 SDK와 API를 활용한 위치 기반 맛집 리뷰 플랫폼입니다. 사용자는 지도 위에서 맛집을 검색하고, 마커를 통해 리뷰가 있는 장소를 확인하며, 직접 리뷰를 작성하고 열람할 수 있습니다. 회원가입 없이 비회원으로 모든 기능을 사용할 수 있어 진입 장벽이 낮고 빠른 사용성을 제공합니다.

### 1.2 제품 비전

"언제 어디서나 빠르게 접근 가능한 간편한 맛집 리뷰 플랫폼"

### 1.3 핵심 가치

- **간편성**: 회원가입 없이 즉시 사용 가능
- **시각성**: 지도 기반 인터페이스로 위치 중심의 직관적인 정보 제공
- **접근성**: 모바일 최적화로 이동 중에도 편리한 사용
- **투명성**: 평점과 리뷰를 통한 신뢰성 있는 정보 공유

### 1.4 제품 범위

**포함 범위**:
- 네이버 지도 SDK를 활용한 지도 표시 및 마커 기능
- 네이버 장소검색 API를 활용한 장소 검색
- 비회원 리뷰 작성 (임시 비밀번호 기반 인증)
- 장소별 리뷰 조회 및 상세 보기
- 평점 시스템 (평균 평점, 리뷰 개수)

**제외 범위** (현재 버전에서는 미포함):
- 회원가입 및 로그인 기능
- 사용자 프로필 관리
- 소셜 로그인 연동
- 리뷰 좋아요/댓글 기능
- 사진 업로드 기능
- 알림 시스템
- 맛집 추천 알고리즘

## 3. 목표 및 성공 지표

### 3.1 비즈니스 목표

1. **시장 검증**: 지도 기반 리뷰 플랫폼에 대한 사용자 수요 확인
2. **사용자 기반 구축**: 출시 후 3개월 내 월간 활성 사용자 1,000명 확보
3. **콘텐츠 축적**: 3개월 내 5,000개 이상의 리뷰 데이터 확보

### 3.2 핵심 성과 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| **MAU** (월간 활성 사용자) | 1,000명 | Google Analytics |
| **일평균 리뷰 작성 수** | 50개 | 데이터베이스 쿼리 |
| **페이지 로딩 속도** | 3초 이내 | Lighthouse 성능 점수 |
| **모바일 사용 비율** | 80% 이상 | Analytics 디바이스 분석 |
| **장소당 평균 리뷰 수** | 3개 이상 | 데이터 분석 |
| **검색 성공률** | 95% 이상 | API 성공/실패 로그 |

### 3.3 사용자 목표

- 원하는 장소를 3번의 터치/클릭 이내에 찾을 수 있어야 함
- 리뷰 작성이 2분 이내에 완료되어야 함
- 지도 로딩이 2초 이내에 완료되어야 함

## 5. 포함 페이지

### 5.1 페이지 구조 개요

총 **3개의 주요 페이지**로 구성되며, 모든 페이지는 모바일 최적화된 MainLayout 내부에 렌더링됩니다.

### 5.2 페이지 상세

#### 5.2.1 홈 페이지 (`/`)

**목적**: 지도 기반 메인 인터페이스 제공, 리뷰가 있는 장소 탐색

**주요 구성 요소**:
- 네이버 지도 SDK 기본 지도 렌더링
- 상단 검색 바 (장소 검색 입력창 + 검색 버튼)
- 지도 영역 내 리뷰 데이터 존재 장소 마커 표시
- 장소 검색 결과 Dialog

**주요 기능**:
1. 지도 표시 및 인터랙션 (확대/축소, 이동)
2. 현재 지도 영역 내 리뷰 있는 장소 마커 표시
3. 장소명 검색 (네이버 장소검색 API)
4. 검색 결과 Dialog 표시
5. 마커 클릭 시 장소 상세 페이지로 이동
6. Dialog 내 '리뷰 작성' 버튼으로 리뷰 작성 페이지 이동

**레이아웃**:
```
+----------------------------------+
|  [검색창]              [검색버튼]  |
+----------------------------------+
|                                  |
|                                  |
|        네이버 지도 영역            |
|        (마커들 표시)               |
|                                  |
|                                  |
+----------------------------------+
```

#### 5.2.2 리뷰 작성 페이지 (`/review/create`)

**목적**: 선택한 장소에 대한 리뷰 작성

**주요 구성 요소**:
- 선택한 장소 정보 카드 (장소명, 주소, 카테고리 등)
- 리뷰 작성 폼
  - 작성자명 입력 필드
  - 평점 선택 (1~5점)
  - 리뷰 본문 입력 필드 (textarea)
  - 임시 비밀번호 입력 필드 (4자리 이상)
- 제출 버튼
- 작성 완료 모달

**주요 기능**:
1. 장소 정보 표시 (네이버 장소검색 API 데이터)
2. 리뷰 데이터 입력 및 유효성 검증
3. 리뷰 저장 (DB에 저장)
4. 저장 완료 후 완료 상태 모달 표시
5. 확인 버튼 클릭 시 홈 페이지로 이동

**레이아웃**:
```
+----------------------------------+
| ← 뒤로가기                        |
+----------------------------------+
| [장소 정보 카드]                   |
| 장소명: OO식당                    |
| 주소: 서울시 강남구...             |
+----------------------------------+
| 작성자명: [________]              |
| 평점: ☆☆☆☆☆                      |
| 리뷰 내용:                        |
| [_________________________]      |
| [_________________________]      |
| 비밀번호(4자리 이상):              |
| [________]                       |
|                                  |
|         [리뷰 작성하기]            |
+----------------------------------+
```

#### 5.2.3 장소 상세 페이지 (`/place/detail`)

**목적**: 선택한 장소의 정보 및 리뷰 목록 조회

**주요 구성 요소**:
- 장소 정보 카드
- 리뷰 통계 (평균 평점, 리뷰 개수)
- 리뷰 목록 (무한 스크롤 또는 페이지네이션)
- 각 리뷰 아이템 (작성자, 평점, 내용 미리보기, 작성일)

**주요 기능**:
1. 장소 정보 표시
2. 평균 평점 계산 및 표시
3. 리뷰 개수 표시
4. 리뷰 목록 조회 (최신순 정렬)
5. 리뷰 아이템 클릭 시 리뷰 상세 페이지로 이동
6. '리뷰 작성하기' 버튼으로 리뷰 작성 페이지 이동

**레이아웃**:
```
+----------------------------------+
| ← 뒤로가기                        |
+----------------------------------+
| [장소 정보 카드]                   |
| 장소명: OO식당                    |
| 주소: 서울시 강남구...             |
+----------------------------------+
| 평균 평점: ★★★★☆ 4.2             |
| 리뷰 개수: 23개                   |
|         [리뷰 작성하기]            |
+----------------------------------+
| [리뷰 1]                         |
| 김민지 ★★★★★                     |
| 정말 맛있어요...                  |
| 2025.10.20                       |
+----------------------------------+
| [리뷰 2]                         |
| 박준호 ★★★★☆                     |
| 괜찮은 맛집...                    |
| 2025.10.19                       |
+----------------------------------+
```

## 6. 사용자 여정

### 6.1 주요 사용자 여정 맵

#### Journey 1: 맛집 검색 후 리뷰 작성

**타겟 유저**: 맛집 탐방러 (김민지 페르소나)

**시나리오**: 새로운 카페를 방문한 후 리뷰를 작성하고 싶음

**여정 단계**:

| 단계 | 페이지 | 사용자 행동 | 시스템 응답 | 감정 상태 |
|------|--------|------------|-----------|----------|
| 1. 진입 | **홈 (`/`)** | 웹사이트 접속 | 지도 로딩 및 표시 | 기대 |
| 2. 검색 | **홈 (`/`)** | 상단 검색창에 "강남 카페" 입력 후 검색 | 장소 목록 Dialog 표시 | 관심 |
| 3. 장소 선택 | **홈 (`/`)** | Dialog에서 원하는 카페 선택, '리뷰 작성' 버튼 클릭 | 리뷰 작성 페이지로 이동 | 적극적 |
| 4. 리뷰 작성 | **리뷰 작성 (`/review/create`)** | 작성자명, 평점, 본문, 비밀번호 입력 후 제출 | 데이터 저장, 완료 모달 표시 | 만족 |
| 5. 완료 | **홈 (`/`)** | 완료 모달에서 확인 버튼 클릭 | 홈 페이지로 이동 | 성취감 |

**소요 시간**: 약 2분

**터치/클릭 횟수**: 6회

#### Journey 2: 지도에서 마커 확인 후 리뷰 열람

**타겟 유저**: 직장인 점심 탐색러 (박준호 페르소나)

**시나리오**: 회사 근처에서 점심 먹을 곳을 찾기 위해 리뷰 확인

**여정 단계**:

| 단계 | 페이지 | 사용자 행동 | 시스템 응답 | 감정 상태 |
|------|--------|------------|-----------|----------|
| 1. 진입 | **홈 (`/`)** | 웹사이트 접속 | 지도 로딩 및 현재 위치 주변 표시 | 급함 |
| 2. 탐색 | **홈 (`/`)** | 지도 이동 및 확대/축소로 주변 확인 | 마커들 실시간 업데이트 | 탐색 중 |
| 3. 마커 선택 | **홈 (`/`)** | 마커 클릭 | 장소 상세 페이지로 이동 | 기대 |
| 4. 리뷰 확인 | **장소 상세 (`/place/detail`)** | 평균 평점 확인 및 리뷰 목록 스크롤 | 리뷰 데이터 표시 | 판단 중 |
| 5. 리뷰 상세 | **리뷰 상세 (미구현)** | 특정 리뷰 클릭 | 리뷰 전문 표시 | 확신 |

## 8. 기능 요구사항

### 8.1 지도 기능 (FR-MAP)

#### FR-MAP-001: 기본 지도 표시
- 네이버 지도 SDK를 사용하여 홈 페이지에 지도 렌더링
- 초기 위치: 서울시청 (위도: 37.5665, 경도: 126.9780)
- 기본 줌 레벨: 17

#### FR-MAP-002: 지도 인터랙션
- 사용자는 지도를 드래그하여 이동할 수 있음
- 사용자는 핀치 줌 또는 줌 컨트롤로 확대/축소 가능
- 줌 컨트롤은 지도 우측 상단에 표시

#### FR-MAP-003: 마커 표시
- 현재 지도 영역 내에 리뷰 데이터가 존재하는 장소에 마커 표시
- 마커는 네이버 지도 기본 마커 스타일 사용
- 마커 개수가 과도할 경우 (50개 이상) 클러스터링 적용 (선택적)

#### FR-MAP-004: 마커 클릭 이벤트
- 마커 클릭 시 해당 장소의 장소 상세 페이지로 이동
- 이동 시 placeId를 URL 파라미터로 전달

### 8.2 검색 기능 (FR-SEARCH)

#### FR-SEARCH-001: 장소 검색 입력
- 홈 페이지 상단에 검색 입력창과 검색 버튼 표시
- 입력창 placeholder: "장소명을 검색하세요"
- 엔터키 입력 또는 검색 버튼 클릭으로 검색 실행

#### FR-SEARCH-002: 네이버 장소검색 API 호출
- 사용자 입력 키워드로 네이버 Local Search API 호출
- API 엔드포인트: `/api/search/local`
- 최대 5개 결과 반환

#### FR-SEARCH-003: 검색 결과 Dialog 표시
- 검색 결과를 모달 Dialog로 표시
- 각 장소 항목:
  - 장소명
  - 주소
  - 카테고리
  - '리뷰 작성' 버튼
- Dialog 외부 클릭 시 닫기

#### FR-SEARCH-004: 검색 에러 처리
- 검색 실패 시 에러 메시지 표시: "검색 결과를 불러올 수 없습니다."
- 결과가 없을 시 메시지: "검색 결과가 없습니다."

### 8.3 리뷰 작성 기능 (FR-REVIEW-CREATE)

#### FR-REVIEW-CREATE-001: 리뷰 작성 폼
- 필수 입력 필드:
  - 작성자명 (최대 20자)
  - 평점 (1~5점, 별점 UI)
  - 리뷰 본문 (최소 10자, 최대 500자)
  - 임시 비밀번호 (4자리 이상, 최대 20자)

#### FR-REVIEW-CREATE-002: 입력 유효성 검증
- 작성자명: 공백 불가, 1~20자
- 평점: 필수 선택
- 리뷰 본문: 10~500자
- 비밀번호: 4~20자
- 유효성 실패 시 필드별 에러 메시지 표시

#### FR-REVIEW-CREATE-003: 리뷰 저장
- 제출 버튼 클릭 시 데이터 저장
- 저장 중 로딩 스피너 표시
- 저장 성공 시 완료 모달 표시
- 저장 실패 시 에러 메시지: "리뷰 작성에 실패했습니다. 다시 시도해주세요."

#### FR-REVIEW-CREATE-004: 작성 완료 처리
- 완료 모달 표시: "리뷰가 작성되었습니다!"
- '확인' 버튼 클릭 시 홈 페이지로 이동

### 8.4 장소 상세 기능 (FR-PLACE-DETAIL)

#### FR-PLACE-DETAIL-001: 장소 정보 표시
- 장소명, 주소, 카테고리 표시
- 네이버 장소검색 API 데이터 활용

#### FR-PLACE-DETAIL-002: 리뷰 통계 표시
- 평균 평점 계산 및 별점으로 표시
- 총 리뷰 개수 표시
- 예: "★★★★☆ 4.2 (23개 리뷰)"

#### FR-PLACE-DETAIL-003: 리뷰 목록 조회
- 최신순으로 리뷰 목록 표시
- 각 리뷰 아이템:
  - 작성자명
  - 평점 (별점)
  - 리뷰 본문 (100자까지 미리보기, 이후 "..." 처리)
  - 작성일 (YYYY.MM.DD 형식)

#### FR-PLACE-DETAIL-004: 리뷰 아이템 클릭
- 리뷰 아이템 클릭 시 리뷰 상세 페이지로 이동 (향후 구현)
- 현재 버전에서는 클릭 시 본문 전체를 보여주는 모달 표시

#### FR-PLACE-DETAIL-005: 리뷰 작성 버튼
- '리뷰 작성하기' 버튼 표시
- 클릭 시 리뷰 작성 페이지로 이동

## 13. 제약사항 및 가정

### 13.1 제약사항

#### 비즈니스 제약사항
1. **비회원 전용**: 회원가입 기능은 구현하지 않음
2. **유료 API**: 네이버 지도 API는 모두 유료 (2025년 7월부터)
3. **모바일 최적화**: 모든 페이지는 360x720 레이아웃에 최적화

#### 기술 제약사항
1. **Next.js 15**: App Router만 사용 (Pages Router 미사용)
2. **네이버 API**: 네이버 클라우드 플랫폼 가입 및 결제 수단 등록 필수
3. **브라우저 호환성**: 최신 Chrome, Safari, Firefox (ES2020+ 지원)

#### 법적/정책 제약사항
1. **개인정보 보호**: 비밀번호는 해싱 후 저장 (bcrypt)
2. **저작권**: 네이버 지도 API 이용 약관 준수
3. **리뷰 책임**: 사용자 작성 리뷰에 대한 면책 조항 필요

### 13.2 가정

#### 사용자 가정
1. 사용자는 모바일 디바이스를 주로 사용한다
2. 사용자는 위치 권한을 허용한다 (선택적)
3. 사용자는 회원가입 없이 사용할 수 있는 간편함을 선호한다

#### 기술 가정
1. 네이버 API는 안정적으로 작동한다
2. Vercel을 사용하여 배포 및 호스팅한다
3. PostgreSQL 데이터베이스는 Vercel Postgres를 사용한다

#### 데이터 가정
1. 초기에는 리뷰 데이터가 적어도 정상 작동한다
2. 장소 ID는 네이버 API에서 제공하는 고유 ID를 사용한다
3. 리뷰는 삭제되지 않는다 (soft delete 미적용)

### 13.3 리스크 및 대응 방안

| 리스크 | 영향도 | 발생 가능성 | 대응 방안 |
|--------|--------|------------|----------|
| 네이버 API 비용 초과 | 높음 | 중간 | API 호출 최적화, 캐싱 적용 |
| 악의적 리뷰 작성 | 중간 | 높음 | Rate Limiting, 신고 기능 추가 (v1.2) |
| 개인정보 유출 | 높음 | 낮음 | 비밀번호 해싱, HTTPS 사용 |
| 성능 저하 (대량 마커) | 중간 | 중간 | 마커 클러스터링, 페이지네이션 |
| 검색 API 장애 | 중간 | 낮음 | 에러 핸들링, 재시도 로직 |
