## 1. 개요

### 1.1 문서 목적

본 문서는 네이버 지도 기반 맛집 리뷰 플랫폼의 사용자 플로우를 정의합니다. 각 페이지별 사용자 입력, 시스템 처리, 출력을 명확히 기술하여 개발 시 참고할 수 있도록 합니다.

### 1.2 플로우 표기 규칙

각 플로우는 다음 형식으로 작성됩니다:

- **입력**: 사용자가 제공하는 모든 UI 입력 및 상호작용
- **처리**: 시스템 내부 로직을 단계별로 기술
- **출력**: 사용자로의 피드백 및 사이드 이펙트
- **예외**: 발생 가능한 엣지 케이스 및 대응

## 3. 페이지별 상세 플로우

### 3.1 홈 페이지 (`/`)

#### 3.1.1 초기 로딩

**입력**
- 사용자가 웹사이트 URL에 접속

**처리**
1. Next.js 페이지 컴포넌트 렌더링 시작
2. `next/script`를 통해 네이버 지도 SDK 스크립트 로드 (Client ID 포함)
3. 스크립트 로드 완료 대기
4. `window.naver` 객체 활성화 확인
5. 네이버 지도 인스턴스 생성
   - 초기 중심 좌표: 서울시청 (37.5665, 126.9780)
   - 줌 레벨: 17
   - 줌 컨트롤: 우측 상단 표시
6. 현재 지도 영역(viewport) 범위 계산
7. 데이터베이스에서 해당 영역 내 리뷰가 존재하는 장소 조회
8. 조회된 장소들의 좌표에 마커 생성 및 지도에 추가

**출력**
- 네이버 지도가 화면에 표시됨
- 리뷰가 있는 장소에 마커가 표시됨
- 상단에 검색창 및 검색 버튼이 표시됨
- 로딩 중 스피너 표시 (지도 로딩 전)

**예외**
- 스크립트 로드 실패: 에러 메시지 표시 및 재시도 버튼 제공
- Client ID 누락: 환경 변수 설정 안내 메시지 표시
- 네트워크 오류: 지도 로딩 실패 메시지 및 새로고침 버튼 표시

---

#### 3.1.2 지도 인터랙션

**입력**
- 사용자가 지도를 드래그하여 이동
- 사용자가 핀치 줌 또는 줌 컨트롤로 확대/축소

**처리**
1. 지도 이동/줌 이벤트 감지
2. 현재 지도 영역(viewport) 범위 재계산
3. 디바운스 적용 (500ms)
4. 데이터베이스에서 새로운 영역 내 리뷰 존재 장소 조회
5. 기존 마커 제거
6. 새로운 마커 생성 및 추가
7. 마커 개수가 50개 초과 시 클러스터링 적용 (선택적)

**출력**
- 지도가 부드럽게 이동/확대/축소됨
- 새로운 영역에 해당하는 마커들이 표시됨
- 영역 밖 마커는 제거됨

**예외**
- 마커 조회 실패: 기존 마커 유지, 콘솔 에러 로깅
- 과도한 마커 (50개 이상): 클러스터링 적용 또는 줌 확대 안내 메시지

---

#### 3.1.3 마커 클릭

**입력**
- 사용자가 지도 위 마커를 클릭

**처리**
1. 마커 클릭 이벤트 핸들러 실행
2. 클릭된 마커의 장소 ID(placeId) 추출
3. Next.js Router를 통해 장소 상세 페이지로 네비게이션
4. URL에 placeId를 쿼리 파라미터로 전달
   - 예: `/place/detail?placeId=12345`

**출력**
- 페이지 전환 애니메이션 (1초 이내)
- 장소 상세 페이지로 이동

**예외**
- placeId 누락: 에러 페이지로 이동 또는 홈으로 리다이렉트
- 네비게이션 실패: 에러 메시지 표시 및 재시도 버튼

---

#### 3.1.4 장소 검색

**입력**
- 사용자가 상단 검색창에 장소명 입력 (예: "강남 카페")
- 사용자가 엔터키를 누르거나 검색 버튼 클릭

**처리**
1. 폼 제출 이벤트 핸들러 실행
2. 입력값 유효성 검증
   - 빈 문자열 체크
   - 트림 처리
3. 로딩 상태 활성화 (버튼 비활성화, 스피너 표시)
4. 네이버 장소검색 API 호출
   - 프론트엔드에서 `/api/search/local?query={검색어}` 호출
   - 백엔드에서 네이버 Local Search API 호출
   - 헤더에 Client ID/Secret 포함
5. API 응답 수신 대기
6. 응답 데이터 파싱 (최대 5개 결과)
7. 검색 결과 Dialog 컴포넌트 렌더링

**출력**
- 로딩 스피너 표시 (검색 중)
- 검색 결과 Dialog 표시
  - 각 장소 항목:
    - 장소명
    - 주소 (도로명 우선, 없으면 지번)
    - 카테고리
    - '리뷰 작성' 버튼
- Dialog 외부 클릭 또는 닫기 버튼으로 닫기 가능

**예외**
- 빈 입력: 에러 메시지 표시 ("검색어를 입력해주세요")
- API 호출 실패: 에러 메시지 표시 ("검색 결과를 불러올 수 없습니다. 다시 시도해주세요.")
- 결과 없음: 안내 메시지 표시 ("검색 결과가 없습니다. 다른 검색어로 시도해보세요.")
- 네트워크 타임아웃: 타임아웃 메시지 및 재시도 버튼

---

#### 3.1.5 검색 결과에서 리뷰 작성

**입력**
- 사용자가 검색 결과 Dialog의 특정 장소 항목에서 '리뷰 작성' 버튼 클릭

**처리**
1. 버튼 클릭 이벤트 핸들러 실행
2. 선택한 장소 정보 추출
   - placeId
   - placeName
   - address
   - category
   - latitude
   - longitude
3. Next.js Router를 통해 리뷰 작성 페이지로 네비게이션
4. URL 쿼리 파라미터에 장소 정보 전달
   - 예: `/review/create?placeId=12345&placeName=카페&address=서울시...`

**출력**
- Dialog 닫힘
- 페이지 전환 애니메이션
- 리뷰 작성 페이지로 이동

**예외**
- 장소 정보 불완전: 필수 필드(placeId, placeName) 누락 시 에러 메시지 표시
- 네비게이션 실패: 에러 메시지 및 홈으로 돌아가기 버튼

---

### 3.2 리뷰 작성 페이지 (`/review/create`)

#### 3.2.1 페이지 로딩

**입력**
- URL 쿼리 파라미터로 장소 정보 수신
  - `placeId` (필수)
  - `placeName` (필수)
  - `address` (선택)
  - `category` (선택)

**처리**
1. Next.js 페이지 컴포넌트 렌더링
2. URL 쿼리 파라미터 파싱
3. 필수 파라미터 검증 (placeId, placeName)
4. 장소 정보 카드 렌더링
5. 리뷰 작성 폼 초기화
   - 작성자명: 빈 문자열
   - 평점: 선택 안 됨
   - 본문: 빈 문자열
   - 비밀번호: 빈 문자열
6. React Hook Form 설정
7. Zod 스키마 검증 설정

**출력**
- 장소 정보 카드 표시
  - 장소명
  - 주소
  - 카테고리 (있는 경우)
- 리뷰 작성 폼 표시
  - 작성자명 입력 필드
  - 평점 선택 UI (별점 1~5)
  - 리뷰 본문 입력 필드 (textarea)
  - 비밀번호 입력 필드
- 제출 버튼 (활성 상태)

**예외**
- 필수 파라미터 누락: 에러 페이지 또는 홈으로 리다이렉트
- 잘못된 파라미터 형식: 에러 메시지 표시 및 홈으로 돌아가기 버튼

---

#### 3.2.2 폼 입력

**입력**
- 사용자가 각 필드에 데이터 입력
  - 작성자명: 텍스트 (최대 20자)
  - 평점: 별점 클릭 (1~5)
  - 본문: 멀티라인 텍스트 (최소 10자, 최대 500자)
  - 비밀번호: 텍스트 (최소 4자, 최대 20자)

**처리**
1. 각 필드 onChange 이벤트 감지
2. React Hook Form 상태 업데이트
3. 실시간 유효성 검증 (onBlur 또는 onChange)
4. 유효성 검증 규칙:
   - 작성자명: 1~20자, 공백만 불가
   - 평점: 1~5 중 하나 선택 필수
   - 본문: 10~500자
   - 비밀번호: 4~20자
5. 검증 결과에 따라 에러 메시지 표시/숨김
6. 모든 필드 유효할 경우 제출 버튼 활성화

**출력**
- 입력값이 실시간으로 필드에 표시됨
- 유효성 검증 실패 시 필드별 에러 메시지 표시
  - 작성자명: "작성자명은 1~20자로 입력해주세요"
  - 평점: "평점을 선택해주세요"
  - 본문: "리뷰는 10~500자로 작성해주세요"
  - 비밀번호: "비밀번호는 4~20자로 입력해주세요"
- 현재 입력 글자 수 표시 (본문, 작성자명)

**예외**
- 특수문자 입력: 허용 (XSS 방지 처리는 서버에서 수행)
- 글자 수 초과: 입력 차단 또는 초과분 자동 제거
- 공백만 입력: 에러 메시지 표시

---

#### 3.2.3 리뷰 제출

**입력**
- 사용자가 '리뷰 작성하기' 버튼 클릭

**처리**
1. 폼 제출 이벤트 핸들러 실행
2. 최종 유효성 검증 (Zod 스키마)
3. 검증 실패 시 에러 메시지 표시 후 중단
4. 검증 성공 시 로딩 상태 활성화
   - 버튼 비활성화
   - 로딩 스피너 표시
5. 리뷰 데이터 객체 생성
   ```typescript
   {
     placeId: string,
     placeName: string,
     address: string,
     authorName: string,
     rating: number,
     content: string,
     password: string, // 평문
   }
   ```
6. 백엔드 API 호출 (`POST /api/reviews`)
7. 백엔드에서 처리:
   - 비밀번호 bcrypt 해싱 (salt rounds: 10)
   - 장소 정보 존재 여부 확인 (없으면 생성)
   - 리뷰 데이터 저장 (PostgreSQL)
   - 장소 통계 업데이트 (reviewCount +1, averageRating 재계산)
   - 트랜잭션 처리
8. 응답 수신 대기
9. 성공 응답 수신 시 완료 모달 표시

**출력**
- 로딩 스피너 표시 (제출 중)
- 성공 시:
  - 완료 모달 표시
  - 제목: "리뷰가 작성되었습니다!"
  - 내용: "소중한 리뷰 감사합니다."
  - 확인 버튼
- 실패 시:
  - 에러 메시지 표시: "리뷰 작성에 실패했습니다. 다시 시도해주세요."
  - 로딩 상태 해제
  - 입력값 유지

**예외**
- 네트워크 오류: 에러 메시지 및 재시도 버튼
- 서버 오류 (500): "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요."
- 데이터베이스 저장 실패: 에러 로깅 및 사용자에게 안내
- 중복 제출 방지: 제출 중 버튼 비활성화 및 추가 클릭 무시

---

#### 3.2.4 작성 완료 후 홈 이동

**입력**
- 사용자가 완료 모달의 '확인' 버튼 클릭

**처리**
1. 버튼 클릭 이벤트 핸들러 실행
2. 모달 닫기
3. Next.js Router를 통해 홈 페이지로 네비게이션
4. 홈 페이지 마운트 시 지도 재렌더링
5. 작성된 리뷰의 장소에 새로운 마커 표시 (또는 기존 마커 유지)

**출력**
- 모달 닫힘
- 페이지 전환 애니메이션
- 홈 페이지 표시
- 작성한 장소에 마커 표시됨

**예외**
- 네비게이션 실패: 에러 메시지 및 재시도 버튼

### 3.3 장소 상세 페이지 (`/place/detail`)

#### 3.3.1 페이지 로딩

**입력**
- URL 쿼리 파라미터로 placeId 수신
  - 예: `/place/detail?placeId=12345`

**처리**
1. Next.js 페이지 컴포넌트 렌더링
2. URL 쿼리 파라미터 파싱
3. placeId 검증 (필수)
4. 로딩 상태 활성화 (스피너 표시)
5. 백엔드 API 호출 (`GET /api/places/{placeId}`)
6. 장소 정보 조회
   - 네이버 API에서 장소 상세 정보 조회
   - 또는 데이터베이스에 저장된 정보 조회
7. 백엔드 API 호출 (`GET /api/reviews?placeId={placeId}`)
8. 해당 장소의 리뷰 목록 조회 (최신순 정렬)
9. 평균 평점 계산
   - 모든 리뷰의 평점 합계 / 리뷰 개수
   - 소수점 첫째 자리까지 표시
10. 리뷰 개수 계산
11. 응답 데이터로 UI 렌더링

**출력**
- 로딩 스피너 표시 (데이터 로딩 중)
- 장소 정보 카드 표시
  - 장소명
  - 주소 (도로명 우선)
  - 카테고리
- 리뷰 통계 표시
  - 평균 평점: 별점 + 숫자 (예: ★★★★☆ 4.2)
  - 리뷰 개수 (예: 23개 리뷰)
- '리뷰 작성하기' 버튼 표시
- 리뷰 목록 표시
  - 각 리뷰 아이템:
    - 작성자명
    - 평점 (별점)
    - 본문 미리보기 (100자까지, 이후 "..." 처리)
    - 작성일 (YYYY.MM.DD 형식)

**예외**
- placeId 누락: 에러 페이지 또는 홈으로 리다이렉트
- 장소 정보 조회 실패: 에러 메시지 표시 ("장소 정보를 불러올 수 없습니다")
- 리뷰 없음: "아직 작성된 리뷰가 없습니다. 첫 번째 리뷰를 작성해보세요!" 메시지 표시
- 네트워크 오류: 에러 메시지 및 재시도 버튼

#### 3.3.2 리뷰 목록 스크롤

**처리**
1. 스크롤 이벤트 감지

3. 리스트 하단 도달 감지 (무한 스크롤 트리거)

## 5. 엣지 케이스 처리

### 5.1 네트워크 및 API 관련

#### 5.1.1 네이버 지도 SDK 로드 실패

**상황**:
- 네이버 클라우드 플랫폼 서버 장애
- 사용자 네트워크 불안정
- Client ID 오류

**처리**:
1. Script 로드 실패 감지 (onError 핸들러)
2. 에러 메시지 표시
   - "지도를 불러올 수 없습니다. 잠시 후 다시 시도해주세요."
3. 재시도 버튼 제공
4. 재시도 클릭 시 페이지 새로고침 또는 스크립트 재로드
5. 3회 연속 실패 시 오류 보고 링크 제공

**사용자 피드백**:
- 에러 UI 컴포넌트 표시 (지도 영역 대체)
- 재시도 버튼
- 고객 지원 링크

---

#### 5.1.2 네이버 장소검색 API 호출 실패

**상황**:
- API 서버 장애
- 인증 정보 오류 (401)
- Rate Limit 초과 (429)
- 네트워크 타임아웃

**처리**:
1. API 호출 try-catch 래핑
2. HTTP 상태 코드별 처리:
   - **400**: "잘못된 검색어입니다. 다시 입력해주세요."
   - **401**: "인증 오류가 발생했습니다. 관리자에게 문의하세요." (콘솔 에러 로깅)
   - **404**: "검색 결과가 없습니다."
   - **429**: "요청이 너무 많습니다. 잠시 후 다시 시도해주세요."
   - **500**: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."
   - **Timeout**: "검색 시간이 초과되었습니다. 다시 시도해주세요."
3. 재시도 버튼 제공 (429, 500, Timeout의 경우)
4. 에러 로깅 (Sentry 등)

**사용자 피드백**:
- Toast 메시지 또는 Dialog로 에러 내용 표시
- 재시도 버튼
- 검색어 유지 (재입력 불필요)

---

#### 5.1.3 리뷰 저장 API 실패

**상황**:
- 데이터베이스 연결 오류
- 트랜잭션 실패
- 유효성 검증 실패 (서버 측)
- 네트워크 중단

**처리**:
1. API 호출 에러 핸들링
2. 서버 응답 에러 메시지 파싱
3. 사용자 친화적 메시지로 변환:
   - DB 오류: "일시적인 오류가 발생했습니다. 다시 시도해주세요."
   - 유효성 오류: 서버에서 반환한 필드별 에러 메시지 표시
   - 네트워크 오류: "네트워크 연결을 확인해주세요."
4. 입력값 유지 (재작성 방지)
5. 재시도 버튼 제공
6. 로컬 스토리지에 임시 저장 (선택적, 개인정보 주의)

**사용자 피드백**:
- 에러 메시지 Alert 또는 Toast
- 재시도 버튼
- 작성한 내용 유지 안내

---

### 5.2 데이터 및 상태 관련

#### 5.2.1 리뷰 없는 장소

**상황**:
- 사용자가 마커를 클릭했으나 해당 장소의 리뷰가 없음 (데이터 정합성 오류)
- 장소 상세 페이지 접근 시 리뷰 개수 0

**처리**:
1. 리뷰 개수 확인 (0개)
2. 빈 상태 UI 표시
   - 메시지: "아직 작성된 리뷰가 없습니다."
   - 서브 메시지: "첫 번째 리뷰를 작성해보세요!"
3. '리뷰 작성하기' 버튼 강조 표시
4. 평균 평점 표시 안 함 (또는 "-" 표시)

**사용자 피드백**:
- 빈 상태 일러스트레이션 표시
- CTA 버튼: "첫 리뷰 작성하기"

---

#### 5.2.2 과도한 마커 (50개 이상)

**상황**:
- 사용자가 지도를 넓게 확대하여 viewport에 50개 이상의 장소 마커 필요

**처리**:
1. 마커 개수 확인
2. 50개 초과 시 클러스터링 적용
   - 네이버 지도 MarkerClustering 라이브러리 사용
   - 또는 줌 레벨에 따라 마커 필터링
3. 클러스터 마커 클릭 시 해당 영역으로 줌 인
4. 또는 "지도를 더 확대해주세요" 안내 메시지 표시

**사용자 피드백**:
- 클러스터 마커 표시 (숫자 포함)
- 줌 안내 Toast 메시지 (선택적)

---

#### 5.2.3 URL 파라미터 누락 또는 잘못된 값

**상황**:
- 리뷰 작성 페이지 접근 시 placeId 누락
- 장소 상세 페이지 접근 시 placeId 누락
- placeId 값이 숫자가 아닌 경우

**처리**:
1. 페이지 로딩 시 URL 파라미터 검증
2. 필수 파라미터 누락 확인
3. 파라미터 타입 검증 (필요 시)
4. 검증 실패 시:
   - 에러 페이지 표시
   - 에러 메시지: "잘못된 접근입니다."
   - 홈으로 돌아가기 버튼
5. 3초 후 자동으로 홈으로 리다이렉트 (선택적)

**사용자 피드백**:
- 에러 페이지 UI
- 명확한 안내 메시지
- 홈으로 돌아가기 버튼

---

#### 5.2.4 중복 제출 방지

**상황**:
- 사용자가 리뷰 작성 폼에서 제출 버튼을 빠르게 여러 번 클릭

**처리**:
1. 제출 버튼 클릭 시 즉시 비활성화
2. 로딩 상태 활성화
3. API 호출 진행 중 추가 클릭 무시
4. 응답 수신 후 버튼 상태 복원 (성공 시 완료 모달, 실패 시 재활성화)

**사용자 피드백**:
- 버튼 비활성화 (회색 처리)
- 로딩 스피너 표시
- 버튼 텍스트 변경: "작성 중..."

---

### 5.3 사용자 입력 관련

#### 5.3.1 특수문자 및 스크립트 입력

**상황**:
- 사용자가 리뷰 본문에 `<script>alert('XSS')</script>` 등 악의적 코드 입력
- 특수문자 과다 입력

**처리**:
1. 프론트엔드: 입력 허용 (사용자 경험 저하 방지)
2. 백엔드: 입력값 sanitize
   - HTML 태그 이스케이프 처리
   - XSS 방지 라이브러리 사용 (예: DOMPurify)
3. 데이터베이스 저장 전 검증
4. 출력 시 dangerouslySetInnerHTML 사용 금지
5. 텍스트로만 렌더링

**사용자 피드백**:
- 입력은 자유롭게 허용
- 저장 후 표시 시 안전하게 렌더링

---

#### 5.3.2 글자 수 초과

**상황**:
- 사용자가 리뷰 본문 500자 제한을 초과하여 입력 시도
- 작성자명 20자 초과

**처리**:
1. 실시간 글자 수 카운팅
2. 제한 초과 시:
   - **방법 1**: 추가 입력 차단 (maxLength 속성)
   - **방법 2**: 입력은 허용하되 제출 시 에러 표시
3. 현재 글자 수 / 최대 글자 수 표시 (예: 485 / 500)
4. 제한 초과 시 빨간색으로 표시
5. 제출 버튼 비활성화

**사용자 피드백**:
- 실시간 글자 수 카운터
- 제한 초과 시 빨간색 경고 텍스트
- 에러 메시지: "본문은 최대 500자까지 입력 가능합니다."

---

#### 5.3.3 빈 값 또는 공백만 입력

**상황**:
- 사용자가 작성자명에 공백만 입력 (예: "   ")
- 본문에 엔터만 여러 번 입력

**처리**:
1. 제출 시 입력값 trim 처리
2. trim 후 빈 문자열 검증
3. 공백만 있는 경우 에러 메시지 표시
4. 필드별 에러:
   - 작성자명: "작성자명을 입력해주세요."
   - 본문: "리뷰 내용을 입력해주세요."

**사용자 피드백**:
- 필드별 에러 메시지 표시
- 해당 필드에 포커스 이동

---

### 5.4 브라우저 및 디바이스 관련

#### 5.4.1 위치 권한 거부 (현재 위치 기능)

**상황**:
- 사용자가 '현재 위치로 이동' 버튼 클릭 시 위치 권한 거부

**처리**:
1. Geolocation API 권한 요청
2. 거부 시 에러 핸들링
3. 안내 메시지 표시:
   - "위치 권한이 필요합니다."
   - "브라우저 설정에서 위치 권한을 허용해주세요."
4. 권한 설정 방법 안내 링크 제공 (선택적)

**사용자 피드백**:
- Toast 메시지 또는 Alert
- 권한 설정 안내

---

#### 5.4.2 오래된 브라우저 (ES2020 미지원)

**상황**:
- 사용자가 구형 브라우저로 접속 (예: IE11)
- JavaScript 최신 문법 미지원

**처리**:
1. 브라우저 감지 (User Agent 또는 Feature Detection)
2. 미지원 브라우저 감지 시:
   - 안내 페이지 표시
   - 메시지: "이 웹사이트는 최신 브라우저에서 사용 가능합니다."
   - 권장 브라우저 목록 표시 (Chrome, Safari, Firefox)
3. 또는 Polyfill 제공 (성능 저하 가능성)

**사용자 피드백**:
- 브라우저 업데이트 안내 페이지
- 권장 브라우저 다운로드 링크

---

#### 5.4.3 화면 크기 제약 (MainLayout 제한)

**상황**:
- 사용자가 데스크톱 큰 화면에서 접속
- MainLayout 최대 너비 360px, 높이 720px 제약

**처리**:
1. CSS로 최대 너비/높이 제한
2. 중앙 정렬 배치
3. 배경 색상 또는 패턴 추가 (선택적)
4. 모바일 최적화 안내 메시지 표시 (선택적)

**사용자 피드백**:
- 모바일 뷰포트 시뮬레이션 표시
- 안내 텍스트: "이 서비스는 모바일에 최적화되어 있습니다."

---

### 5.5 데이터 정합성 관련

#### 5.5.1 리뷰와 장소 데이터 불일치

**상황**:
- 리뷰는 존재하나 장소 정보가 DB에 없음
- 장소 정보와 네이버 API 정보 불일치

**처리**:
1. 장소 상세 페이지 로딩 시 장소 정보 조회
2. DB에 없을 경우 네이버 API로 실시간 조회
3. 조회된 정보 DB에 저장 (캐싱)
4. API 조회 실패 시:
   - 최소한의 정보만 표시 (placeId, placeName)
   - 리뷰는 정상 표시
   - 주소/카테고리는 "정보 없음" 표시

**사용자 피드백**:
- 정상적인 UI 표시 (에러 최소화)
- 정보 없는 필드는 "-" 또는 "정보 없음" 표시

---

#### 5.5.2 평균 평점 계산 오류

**상황**:
- 리뷰 삭제 후 평균 평점 미갱신 (향후 삭제 기능 추가 시)
- 소수점 계산 오류

**처리**:
1. 평균 평점 실시간 계산
   - 모든 리뷰 평점 합계 / 리뷰 개수
   - 소수점 첫째 자리 반올림
2. 리뷰 개수 0인 경우 평점 표시 안 함
3. 계산 로직 백엔드에서 수행 (프론트엔드는 표시만)
4. 트랜잭션으로 리뷰 저장과 평점 갱신 동시 처리

**사용자 피드백**:
- 정확한 평점 표시
- 리뷰 없을 시 "-" 또는 "평점 없음"
